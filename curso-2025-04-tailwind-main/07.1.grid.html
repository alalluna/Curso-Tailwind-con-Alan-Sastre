¿Qué es CSS Grid?

Grid es un sistema de layout bidimensional: te deja colocar elementos en filas y columnas, con control total de cada celda. Tailwind 4.1 traduce cada propiedad de Grid a una clase utilitaria. Esta guía resume todas las utilidades de Grid en Tailwind, con ejemplos listos para copiar.

1. Activar Grid — grid / inline-grid

Las clases que convierten un contenedor en Grid:

  • grid → display: grid;
  • inline-grid → display: inline-grid;

2. Columnas — grid-template-columns

Define cuántas columnas y su tamaño. Tailwind provee:

  • grid-cols-n: n columnas iguales (1–12).
  • grid-cols-none: sin track explícito.
  • Fracciones arbitrarias: [grid-cols-[1fr_2fr_1fr]].

4. Flujo automático — grid-auto-flow

Controla cómo se colocan los ítems sin posición explicita:

  • grid-flow-row (default)
  • grid-flow-col
  • grid-flow-dense
  • grid-flow-row-dense / grid-flow-col-dense

5. Tamaños automáticos — grid-auto-cols / grid-auto-rows

El tamaño asignado a tracks creados automáticamente:

  • auto-cols-{size}
  • auto-rows-{size}

6. Posicionamiento — grid-column / grid-row

Span, start y end:

  • col-span-n / row-span-n
  • col-start-n / row-start-n
  • col-end-n / row-end-n

7. Separación — gap

Espacio entre tracks (filas y/o columnas):

  • gap-n
  • gap-x-n / gap-y-n

11. Subgrid — grid-cols-subgrid / grid-rows-subgrid

Permite que un grid anidado (hijo) adopte las definiciones de track (columnas o filas) de su grid padre. Esto es útil para alinear ítems a través de grids anidados.

  • grid-cols-subgrid: El grid hijo usa las tracks de columna del padre en las que está posicionado.
  • grid-rows-subgrid: El grid hijo usa las tracks de fila del padre en las que está posicionado.

Ejemplo:

Ítem Padre 1 (ocupa 2 cols)
Ítem Padre 2
Ítem Padre 3
Sub-ítem A (alineado con col 1 padre)
Sub-ítem B (alineado con cols 2-3 padre)
Sub-ítem C (alineado con col 4 padre)
<div class="grid grid-cols-4 gap-4">
  <!-- ... ítems padre ... -->
  <div class="col-span-4 grid grid-cols-subgrid gap-4">
    <!-- Los ítems aquí se alinean con las 4 columnas del grid padre -->
    <div class="col-span-1">A</div>
    <div class="col-span-2">B</div>
    <div class="col-start-4">C</div>
  </div>
</div>

12. Valores Arbitrarios y Funciones Avanzadas

Para definiciones de grid más complejas que las utilidades estándar, puedes usar la notación de corchetes [...]. Esto te permite usar cualquier valor CSS válido, incluyendo funciones como minmax(), repeat(), fit-content(), o definir áreas de grid nombradas.

Funciones minmax() y repeat()

Define tamaños flexibles y patrones repetitivos.

  • [grid-cols-[repeat(auto-fit,minmax(150px,1fr))]]: Columnas que se ajustan automáticamente, cada una con un mínimo de 150px y un máximo de 1fr.
  • [grid-rows-[auto_1fr_auto]]: Una fila superior e inferior de tamaño automático, y una central que ocupa el espacio restante.

Ejemplo con repeat(auto-fit, minmax(100px, 1fr)):

<div class="grid [grid-cols-[repeat(auto-fit,minmax(100px,1fr))]] gap-4">
  <!-- ... ítems ... -->
</div>

Áreas de Grid Nombradas (grid-template-areas)

Puedes definir un layout complejo nombrando áreas y asignando ítems a esas áreas usando valores arbitrarios.

Ejemplo:

Header
Sidebar
Main
Footer
<div class="grid [grid-template-areas:'header_header'_'sidebar_main'_'footer_footer'] 
             [grid-template-columns:150px_1fr] 
             [grid-template-rows:auto_1fr_auto] 
             gap-4 h-64">
  <div class="[grid-area:header]">Header</div>
  <div class="[grid-area:sidebar]">Sidebar</div>
  <div class="[grid-area:main]">Main</div>
  <div class="[grid-area:footer]">Footer</div>
</div>

13. Orden de los Ítems — order-*

Aunque Grid define un orden bidimensional, puedes cambiar el orden visual de los ítems (su orden de pintado y tabulación) independientemente de su posición en el DOM, usando las utilidades order-*. Por defecto, todos los ítems tienen order-0.

  • order-n (donde n es 1-12)
  • order-first (order: -9999;)
  • order-last (order: 9999;)
  • order-none (order: 0;)
1. DOM (order-last)
2. DOM (order-0)
3. DOM (order-first)
4. DOM (order-2)
5. DOM (order-1)
<div class="grid grid-cols-3 gap-4">
  <div class="order-last">1</div>
  <div>2</div>
  <div class="order-first">3</div>
  <div class="order-2">4</div>
  <div class="order-1">5</div>
</div>

14. Diseño Responsivo

Todas las utilidades de Grid se pueden aplicar condicionalmente en diferentes tamaños de pantalla usando los prefijos responsivos de Tailwind (como sm:, md:, lg:, xl:, 2xl:).

Por ejemplo, puedes tener un grid de una columna en móviles y de tres columnas en pantallas medianas y mayores.

Ejemplo:

Col 1 (Móvil) / Col 1 (MD)
Col 1 (Móvil) / Col 2 (MD)
Col 1 (Móvil) / Col 3 (MD)
Col 1 (Móvil) / Cols 1-2 (MD)
Col 1 (Móvil) / Col 3 (MD)
<div class="grid grid-cols-1 md:grid-cols-3 gap-4">
  <div>Item 1</div>
  <div>Item 2</div>
  <div>Item 3</div>
  <div class="md:col-span-2">Item 4 (ocupa 2 cols en MD+)</div>
  <div>Item 5</div>
</div>
